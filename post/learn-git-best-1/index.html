Git从入门到进阶(一)

文章目录

前情提要：本文旨在让初学者快速上手，想了解更多细节请查阅 Git官方文档 以及自行Google其他教程。

Git简介与安装

Git是目前世界上使用最广泛的分布式版本控制系统，没有之一。好，废话不多说，让我们开始。

安装，只举几个例子：

  • Ubuntu/Debian: sudo apt-get install git
  • MacOS : brew install git
  • Windows: 访问 官方网站 下载
    • git version查看是否安装成功
    • 安装出问题，请自行搜索

然后配置以下两行：

git config --global user.name 你的英文名

git config --global user.name 你的邮箱

GitHub 全球最大的同性交友平台

GitHub是世界上开发者最多的聚集地，你一定听说过它：

Git就是打开GitHub的钥匙，想要一起来同性交友么？让我们先学会使用Git。

Git本地仓库

常用基础命令:

git init会创建.git目录，即本地仓库。(init即初始化)

git add 路径会把你指定路径的变动提交到暂存区，路径可以是绝对路径、相对路径、.和*。

.gitignore描述哪些变动是不需要提交的。（常见有.DS_Store、.idea、.vscode等）

git commit -m 字符串把暂存区的变动提交到本地仓库，字符串即提交理由,字符串里如果有空格，就要用引号""包起来。

git log用来查看之前的提交历史。

git reflog用来查看所有历史。

git branch X创造X分支。

git checkout X切换到X分支。

git merge将另一个分支合并到当前分支，合并分支有可能会产生冲突。

git status -sb查看哪个/哪些文件冲突，删除掉文件中的====、>>>>、<<<<这些标记，解决冲突。

git branch -d X删除掉无用分支X。

一些细节：

  • .git目录就是本地仓库。
    • 它不会重复复制相同的文件。
    • 它可以支持多个分支。
  • git add处理的是文件变化，而不是文件，比如你删除一个文件后，依然要用git add来添加到暂存区。
  • git commit 不加-m参数默认进入vim编辑器，如遇到请搜索「如何退出Vim」，当然你也可以配置其他的编辑器为默认编辑器。
